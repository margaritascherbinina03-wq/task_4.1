## git checkout

позволяет переключаться между отдельными ветками репозитория. Каждое переключение обновляет файлы в рабочем каталоге на основе данных, которые хранятся в выбранной ветке. При этом каждый последующий коммит автоматически добавляется в активную ветку, выбранную ранее с помощью команды checkout.

## Команда также может выполнять другие задачи, например:

создавать новые ветки в момент переключения;

сбрасывать существующие ветки до состояния других веток;
переключаться на конкретный коммит из локальной или удалённой ветки.

## Базовый синтаксис: git checkout [наименование ветки | хэш коммита]


## -b 

создаёт новую ветку и сразу переключается на неё. Например, git checkout -b new-branch.

## -f 

принудительно перезаписывает локальные изменения, которые не были зафиксированы. Этот флаг используется только в крайних случаях.

## -p 

 позволяет просмотреть и выбрать конкретные изменения для восстановления.
