## git flow

модель ветвления для системы контроля версий Git, которая позволяет структурировать работу с кодом, создавая отдельные ветки для новых функций, исправлений ошибок и релизов. Введена Винсентом Дриссеном в 2010 году.

Основа модели — две основные ветки: 

## master
 — содержит стабильную версию кода, готовую к выпуску.

## develop 
— служит веткой для разработки, где собираются и интегрируются все изменения из вспомогательных веток
